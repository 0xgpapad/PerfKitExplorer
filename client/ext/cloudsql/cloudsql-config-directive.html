Cloud SQL Settings
Provide the username/password that will be used to authenticate to Cloud SQL. Each Cloud SQL administrator is responsible for authorizing this account. Please ensure that you only grant SELECT access to the relevant tables/views. Providing access beyond that can allow dashboard authors to modify your data and/or schemas. You can read more about how to create users and grant access in the MySql documentation, and the script below illustrates how to create and authorize the user to a database (all objects) and table.
CREATE USER '{{ ngModel.username }}'@'localhost' IDENTIFIED BY '{{ ngModel.password }}';
GRANT SELECT ON db1.* TO '{{ ngModel.username }}'@'localhost';
GRANT SELECT ON db2.invoice TO '{{ ngModel.username }}'@'localhost';
